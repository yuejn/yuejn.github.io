Remotely connecting to your Raspberry Pi 3 Model A+

Published: Dec 2018 ⋄ 3 min read
tech

The Raspberry Pi 3 Model A+ is pretty cool. It’s a lot more compact than its predecessor and it comes with wireless and Bluetooth capabilities. This means goodbye ethernet cables and extra dongles!

This is a reference guide on how I initially set up my Raspberry Pi, enabling SSH access so I could connect to it remotely from another computer on the same Wi-Fi network. I had to install a bunch of packages and dependencies onto the Pi and wanted to do that remotely on my laptop.

Assumptions

You have a:

  • Raspberry Pi 3 Model A+

  • Compatible micro SD card (4GB +)

  • Display monitor with HDMI cable

  • USB-powered keyboard

  • Good quality USB-powered power supply

Instructions

  1. Write the Raspbian Lite image to the SD card using Etcher.

  2. After flashing the card, eject and re-insert the SD so you can see boot, the FAT32 partition. (On a Mac? You might not be able to see this partition. Check out extFS for Mac.)

  3. On this boot partition, create a file named ssh. When the Raspberry Pi first boots, it will enable SSH and delete the file.

  4. Connect the Pi to a power supply, display monitor, and a keyboard.

  5. Switch the power supply on and wait for the monitor to start talkin' to you.

  6. Once it stops spitting out information, it will ask you for credentials. The default username is pi with the password raspberry.

  7. Now that we’re in, we’re going to start by connecting to the Wi-Fi, by running sudo raspi_config. A graphical user interface will pop up. Navigate down to Network Options and go through the process. SSID is the name of your Wi-Fi network. (It is very important to set the Wi-Fi country or the Wi-Fi will not work.)

  8. Exit out of the configuration application. Check to see if the Wi-Fi connection is successful with: sudo iwgetid.

  9. Now let’s check what the Raspberry Pi’s static IP address is. This is what we’ll use to SSH into. Type: sudo ifconfig and hit enter. The IP address belonging to the Pi is the value of inet_addr for the wlan0 entry. With this IP address, you should now be able to SSH into the Pi remotely – as long as you’re in the same Wi-Fi network, and the Pi is switched on. :)

ssh pi@172.43.75.232
Password: raspberry

Change the username and password

When you first login to the Pi, you can change the password by entering passwd and hitting enter. You’ll be asked to type in the current password (raspberry) before being asked for a new password, and then confirmation.

For more, check out the documentation on user management on the Raspberry Pi website.
